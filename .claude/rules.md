# このプロジェクトでのルール

## 勝手に紛らわしいことをしない

- **依頼されたこと以外のファイルを作らない。** ユーザーが「〇〇の README を書いて」と言ったときは、その〇〇用の README だけを書く。別の説明用ファイルなどを勝手に作らない。
- **紛らわしいこと・意味が伝わりにくいことを勝手にしない。** 依頼と違う場所に書く、依頼と違う種類のファイルを増やす、といったことはしない。
- ファイルの作成・編集は、**事前に何をどこに書くか（または作るか）を伝え、了承を得てから**行う。

---

## 意見を求められたとき

**ユーザーが意見・考えを求めているときは、作業（ファイルの編集・置換・追加など）をせず、意見だけを述べること。**

- 「どう思う？」「あんたはどう思うの？」「〜でどう？」など、判断や考えを聞かれているときは、回答は意見のみにする。
- 賛成・反対や「こうした方がよい」などの見解を述べ、その場でファイルの変更や修正は行わない。
- 変更してほしいと明示されたときだけ、作業を行う。
