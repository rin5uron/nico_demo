# `docs` ディレクトリ運用ルール

このディレクトリは、ニコ酒店様Webサイト制作プロジェクトに関するすべてのドキュメントを管理する場所です。
情報は以下のルールに従って、各ディレクトリに格納してください。

---

## ディレクトリ構成

```
docs/
├── contacts/          # クライアント関連すべて
│   ├── client_meetings.md    # 対面ヒアリング・訪問記録
│   ├── client_messages.md    # LINE等のメッセージやり取り記録
│   └── proposals/            # 提案書・見積書・契約書（日付別）
├── media/             # ドキュメント用メディアファイル
│   ├── samples/       # スクリーンショット・参考画像
│   └── notes/         # 図解・メモ用画像
├── notes/             # 個人メモ・調査記録
├── spec/              # サイト仕様定義
├── process.log        # 実装変更の技術記録（時系列）
└── README.md          # このファイル
```

---

## 各ファイル・ディレクトリの説明

### `contacts/`
**目的:** クライアント関連のすべての記録・書類を保管します。

| ファイル・ディレクトリ | 内容 |
|---------|------|
| `client_meetings.md` | 対面訪問・ヒアリング記録 |
| `client_messages.md` | LINE・メール等のやり取り記録 |
| `proposals/` | 提案書・見積書・契約書（日付別フォルダで管理） |

---

### `media/`
**目的:** ドキュメント内で参照する画像・メディアファイルを保管します。

| ディレクトリ | 内容 |
|-------------|------|
| `samples/` | サイトのスクリーンショット、デザイン参考画像 |
| `notes/` | 図解・メモ用の画像 |

---

### `notes/`
**目的:** プロジェクト進行中の個人的なメモや調査記録を自由に書き留めます。

**主な内容:**
- 技術調査のメモ
- 参考記事のURL
- アイデアの断片
- 設計検討メモ

---

### `spec/`
**目的:** Webサイトの仕様を定義するドキュメントを保管します。

**主な内容:**
- 要件定義書
- サイトマップ
- ワイヤーフレーム
- 機能一覧
- デザイン仕様書
- 画像命名規則

---

### `process.log`
**目的:** 実装した変更内容を時系列で記録します。

**記録内容:**
- 実装した機能・デザインの変更
- レイアウト調整の詳細
- 技術的な判断と理由
- 参考画像へのリンク

**記録形式:**
```markdown
2025-XX-XX
- 変更内容のサマリー
  - 詳細な変更点
  - 参考: docs/media/samples/XXXXXX.png
```
